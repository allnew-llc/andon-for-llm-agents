  <name>Task 1: Create gotcha-stats.sh with hit rate, staleness, and effectiveness metrics</name>
  <files>skills/tps-kaizen/scripts/gotcha-stats.sh</files>
  <action>
Create `skills/tps-kaizen/scripts/gotcha-stats.sh` following the same POSIX-only bash conventions as `aggregate-incidents.sh` (no jq, no Python — use awk for JSON field extraction and string processing).

**Script structure:**

1. **Header and usage**: Apache 2.0 license header, `set -euo pipefail`, `--help` flag handling. Document env vars: `CLAUDE_PLUGIN_DATA`, `ANDON_STATE_DIR`, and `GOTCHA_DIR` (defaults to `gotchas/` relative to repo root detected via `git rev-parse --show-toplevel` or fallback to script's `../../../../gotchas`).

2. **Resolve data directories**:
   - Incidents dir: Same logic as aggregate-incidents.sh (`${CLAUDE_PLUGIN_DATA}/kaizen/incidents/` or `${ANDON_STATE_DIR:-~/.claude/state}/kaizen/incidents/`)
   - Gotchas dir: `${GOTCHA_DIR:-$(git rev-parse --show-toplevel 2>/dev/null)/gotchas}` with fallback to script-relative path
   - History dir: Same base as incidents but `kaizen/history/` for closed ANDON archives

3. **Load Gotcha registry**: For each `*.yaml` file in gotchas dir:
   - Extract `id` field using the same `extract_json_value`-style awk approach but adapted for YAML (key: "value" or key: >). Use a `extract_yaml_value()` function that handles both `key: "value"` and `key: value` forms.
   - Extract `name` field
   - Extract `pattern` field (handle multi-line `>` folded scalar by reading continuation lines that are indented)
   - Store in parallel arrays: `gotcha_ids[]`, `gotcha_names[]`, `gotcha_patterns[]`

4. **Load incidents**: For each `INC-*` directory:
   - Extract `output_snippet` from `evidence.json` using `extract_json_value`
   - Extract `opened_at` from `evidence.json`
   - Try to find matching close record: scan `kaizen/history/andon-closed-*.json` files for ones whose nested `andon.incident_id` matches this incident's ID. Extract `closed_at` if found.

5. **Compute hit rates (METRIC-01)**: For each Gotcha, count how many incidents match it:
   - Extract significant keywords from the Gotcha's pattern (words 3+ chars, lowercase, exclude common English stopwords — same list as the Python surfacer uses)
   - For each incident's `output_snippet`, check what fraction of significant pattern keywords appear (case-insensitive)
   - If 40%+ of keywords match, count it as a hit (consistent with the surfacer's PARTIAL match threshold)
   - Also count EXACT matches (full normalized pattern is substring of normalized output_snippet)
   - Track which incidents matched which Gotchas (for effectiveness calculation)

6. **Flag stale Gotchas (METRIC-02)**: Any Gotcha with 0 hits gets labeled `** POTENTIALLY STALE **` in the output.

7. **Compute prevention effectiveness (METRIC-03)**:
   - For incidents WITH at least one Gotcha match: calculate average resolution time (closed_at - opened_at). If closed_at not found, mark as "open/unknown".
   - For incidents WITHOUT any Gotcha match: calculate average resolution time.
   - Show both averages and a simple delta (faster/slower/no data).
   - This is a heuristic measure — document that it shows correlation, not causation.

8. **Output format**: Print a structured report:
   ```
   === Gotcha Effectiveness Report ===
   Data source: {incidents_dir}
   Registry: {gotchas_dir}
   Total Gotchas: N
   Total incidents scanned: N
   Period: YYYY-MM-DD to YYYY-MM-DD

   --- Hit Rate Table ---
   ID            | Name                          | Hits | Effectiveness
   GOTCHA-001    | Human Error Stop              |    3 | avg 12min (with) vs 25min (without)
   GOTCHA-002    | 4-Artifact Close Shortcut     |    0 | ** POTENTIALLY STALE **
   ...

   --- Stale Gotchas (zero hits) ---
   GOTCHA-002: 4-Artifact Close Shortcut (discovered: 2026-03-19)
     Consider: broadening pattern or archiving if no longer relevant

   --- Prevention Effectiveness Summary ---
   Incidents with Gotcha match: N (avg resolution: Xmin)
   Incidents without Gotcha match: N (avg resolution: Ymin)
   Delta: Gotcha-matched incidents resolved Z% faster/slower
   Note: Correlation only. Gotcha surfacing may help but is not the sole factor.

   === End of Report ===
   ```

**Implementation notes:**
- Reuse `extract_json_value()` from aggregate-incidents.sh pattern (copy it, do not source the other script)
- Create a new `extract_yaml_value()` function for YAML fields. For the multi-line `pattern` field (uses `>` folded scalar), read subsequent indented lines and join them with spaces.
- Stopwords list: use a minimal hardcoded set matching the Python surfacer's list (the, a, an, is, was, for, of, by, with, that, this, it, not, but, and, or, in, on, at, to, are, were, be, been, has, have, had, do, does, from, so, than, they, them, their, there, what, when, where, which, who, will, you, your, we, our, up, out, per, too, some, into, as, its, no, he, her, him, his, how, if, us)
- Time calculation: use `date` command to convert ISO timestamps to epoch seconds. On macOS, use `date -j -f "%Y-%m-%dT%H:%M:%S" "$ts" "+%s"` (strip trailing Z/timezone first). Handle both GNU and BSD date gracefully with a wrapper function.
- Make the script executable: `chmod +x`
- If no incidents exist or no Gotchas exist, print informative message and exit 0 (same pattern as aggregate-incidents.sh)
