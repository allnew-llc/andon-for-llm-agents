Create skills/tps-kaizen/scripts/ directory and write aggregate-incidents.sh.

The script must:

1. **Resolve data directory**: Check CLAUDE_PLUGIN_DATA env var first — if set, use ${CLAUDE_PLUGIN_DATA}/kaizen/incidents/. If not set, fall back to ${ANDON_STATE_DIR:-$HOME/.claude/state}/kaizen/incidents/. Print the resolved path to stderr for transparency.

2. **Handle empty data**: If directory does not exist or contains no INC-* subdirectories, print "No incident data found in {path}" and exit 0.

3. **Scan incidents**: For each INC-* directory, read analysis.json. Extract: cause_id, category, confidence, root_cause_summary. Use grep/sed/awk for JSON extraction (no jq dependency — OSS portability). Accept that parsing is best-effort for simple flat keys. Also extract opened_at from evidence.json for timeline context.

4. **Group by pattern**: Aggregate incidents by cause_id. For each unique cause_id, count occurrences, list dates, and show the most recent root_cause_summary.

5. **Output format** (plain text, designed for terminal reading):
```
=== Incident Pattern Summary ===
Data source: /path/to/incidents/
Total incidents: N
Unique patterns: M
Period: YYYY-MM-DD to YYYY-MM-DD

--- Recurring Patterns (2+ occurrences) ---

  [cause_id] (N occurrences)
  Category: [category]
  Last seen: YYYY-MM-DD
  Root cause: [root_cause_summary]

--- Single Occurrences ---

  [cause_id]
  Category: [category]
  Date: YYYY-MM-DD
  Root cause: [root_cause_summary]

=== End of Summary ===
```

6. **Script header**: Include shebang `#!/usr/bin/env bash`, `set -euo pipefail`, Apache 2.0 copyright header for AllNew LLC, and a usage comment explaining arguments (none required) and env vars (CLAUDE_PLUGIN_DATA, ANDON_STATE_DIR).

7. **Portability**: No bashisms beyond what bash 3.2+ supports. No jq, no python. Use only POSIX-compatible tools (grep, sed, awk, sort, uniq, wc). Test-friendly: support `--help` flag that prints usage and exits 0.

Make the file executable (chmod +x).

Write quality-trend.sh in skills/tps-kaizen/scripts/.

The script must:

1. **Resolve data directory**: Check CLAUDE_PLUGIN_DATA env var first — if set, use ${CLAUDE_PLUGIN_DATA}/qc/. If not set, fall back to searching for quality-self-assessment-*.json in the current working directory's docs/pipeline/ as a secondary source. Print the resolved path to stderr.

2. **Handle empty data**: If no quality-self-assessment JSON files found, print "No quality assessment data found" and exit 0.

3. **Scan assessments**: Find all quality-self-assessment-*.json files. For each, extract: phase_id, timestamp (or derive from filename/file mtime), overall status (pass/warn/fail), and count of criteria by status. Use grep/sed/awk for JSON extraction (no jq dependency).

4. **Output format** (time-series table for terminal):
```
=== Quality Trend Report ===
Data source: /path/to/qc/
Assessments found: N

Date       | Phase   | Overall | Pass | Warn | Fail
-----------+---------+---------+------+------+-----
YYYY-MM-DD | phase-1 | PASS    |    5 |    0 |    0
YYYY-MM-DD | phase-2 | WARN    |    3 |    2 |    0
YYYY-MM-DD | phase-3 | FAIL    |    2 |    1 |    2

Summary: X PASS, Y WARN, Z FAIL across N assessments
Trend: [improving / stable / degrading] (based on last 3 assessments)

=== End of Report ===
```

5. **Trend detection**: Compare the last 3 assessments' overall status. If latest is better than earliest, say "improving". If same, say "stable". If worse, say "degrading". Order: PASS > WARN > FAIL.

6. **Script header**: Same conventions as aggregate-incidents.sh — shebang, set -euo pipefail, Apache 2.0, usage comment, --help flag.

7. **Portability**: Same constraints — bash 3.2+, no jq, no python, POSIX tools only.
