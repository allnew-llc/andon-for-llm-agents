  <action>
Write five-whys-validator.sh in skills/tps-kaizen/scripts/.

The script must:

1. **Input**: Accept a file path as argument, or read from stdin if no argument given. Print usage on --help and exit 0.

2. **Detect format**: The validator supports two Five Whys formats:
   - **Table format**: Lines matching `| [0-9]+ |` with at least 3 pipe-delimited columns (number, why, verification)
   - **Expanded format**: Sections starting with `### Why [0-9]` containing `**Verification**:` lines

3. **Validate depth (>=5 levels)**:
   - Table format: Count rows where column 1 is a number >= 1
   - Expanded format: Count `### Why N` headings
   - Report the number of levels found. PASS if >= 5, FAIL if < 5.
   - Note: Per Five Whys rules, stopping at 3 is valid if root cause is found. But the validator checks mechanical completeness — the user decides if stopping early is justified.

4. **Validate verification column**:
   - Table format: For each numbered row, check that column 3 (verification) is not empty, not just whitespace, and not a placeholder like `[file/command checked]`, `TBD`, `TODO`, `N/A`
   - Expanded format: For each `### Why N` section, check that `**Verification**:` exists and has non-empty content on the same or next line
   - Report which levels have empty/missing verifications

5. **Check for "human error" stop** (warning, not failure):
   - Scan the last Why's text for phrases like "human error", "I made a mistake", "forgot to", "didn't remember"
   - If found, emit a WARNING: "Final cause may be stopping at human error — consider digging into process/system issues (see Gotchas: Human Error Stop)"

6. **Output format**:
```
=== Five Whys Validation Report ===
File: [path or stdin]

Depth Check:
  Levels found: N
  Required: 5
  Status: PASS / FAIL

Verification Check:
  Level 1: PASS / FAIL (empty verification)
  Level 2: PASS / FAIL (placeholder text)
  ...
  Filled: X/N
  Status: PASS / FAIL

Warnings:
  - [any human-error-stop warnings]

Overall: PASS / FAIL
Reason: [if FAIL, list specific failures]

=== End of Report ===
```

7. **Exit codes**: 0 = PASS, 1 = FAIL, 2 = usage error (no input)

8. **Script header**: shebang `#!/usr/bin/env bash`, `set -euo pipefail`, Apache 2.0 copyright header, usage comment.

9. **Portability**: bash 3.2+, no jq, no python, POSIX tools only (grep, sed, awk).

Make the file executable (chmod +x).
