YAML frontmatter:
- name: standup
- description: "Use when: daily standup, status update, progress report, what did I do yesterday, daily summary, end of day recap. Aggregates git log and task state into a formatted daily summary."
- version: 1.0.0

Content sections:
1. Title: `/standup` — Daily Summary Generator
2. Opening: aggregates git commits (last 24h) and GSD task state into a formatted standup report
3. "When to Use This Skill" table: daily standup, end-of-day summary, progress check, team update
4. Usage: `/standup` (default: last 24h), `/standup 48h` (custom timeframe), `/standup week` (last 7 days)
5. Behavior:
   - Step 1: Run `scripts/standup.sh` to collect raw data (git log + STATE.md)
   - Step 2: Claude reads the script output and formats it into a structured daily summary
   - Step 3: Output format:
     ```
     ## Daily Standup — YYYY-MM-DD
     ### Completed
     - [commit summaries grouped by topic]
     ### In Progress
     - [from STATE.md current position]
     ### Blockers
     - [from STATE.md blockers, or "None"]
     ### Next
     - [from ROADMAP.md next phase/plan]
     ```
6. Gotchas section with 3 named ### patterns:
   - **Timezone Sensitivity**: `git log --since` uses local timezone. If committing across timezones, some commits may be missed or double-counted. Always verify the time range in output.
   - **Shallow Clone Limitation**: In CI environments with shallow clones, git log may not have full history. The script detects shallow repos and warns.
   - **Multi-Repo Projects**: standup.sh runs in the current repo only. For multi-repo projects, run in each repo and consolidate manually or specify paths.

**File 2: `skills/standup/scripts/standup.sh`**

Bash script following Phase 7 patterns:
- `#!/bin/bash` + `set -euo pipefail`
- Apache 2.0 license header
- `--help` flag support
- Accept optional argument: hours (default 24), or "week" (168 hours)
- POSIX tools only (git, awk, sed, date, cat)

Script behavior:
1. Parse arguments (hours or "week")
2. Compute the `--since` value for git log
3. Run `git log --oneline --since="$SINCE" --no-merges` and format output
4. If `.planning/STATE.md` exists in repo root or parent directories, extract:
   - Current position (Phase N of M)
   - Status line
   - Blockers section
5. If `.planning/ROADMAP.md` exists, extract the next incomplete phase
6. Print structured output to stdout:
   ```
   === GIT ACTIVITY (last Nh) ===
   [commit lines]

   === TASK STATE ===
   Position: Phase X of Y (name)
   Status: [status]
   Blockers: [blockers or "None"]

   === NEXT UP ===
   [next phase from roadmap]
   ```
7. Handle edge cases: no commits found, no STATE.md, not a git repo (exit with informative message)

Make the script executable: `chmod +x skills/standup/scripts/standup.sh`
