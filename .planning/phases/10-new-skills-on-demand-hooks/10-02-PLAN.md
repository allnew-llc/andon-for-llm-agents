<!-- On-demand hook pattern established by freeze in Phase 9 -->

Pattern:
1. Hook script (e.g., freeze-guard.sh) permanently registered in ~/.claude/settings.json
2. State file (e.g., ~/.claude/state/freeze-state.json) controls activation
3. When state file absent: hook exits immediately with `echo "{}"` (zero overhead)
4. When state file present: hook enforces rules and denies with JSON response
5. Activation: `/skill on` creates state file
6. Deactivation: `/skill off` deletes state file (rm -f)

Hook deny response format:
```json
{
  "hookSpecificOutput": {
    "hookEventName": "PreToolUse",
    "permissionDecision": "deny",
    "permissionDecisionReason": "REASON: explanation of what was blocked and how to proceed"
  }
}
```

Hook registration in settings.json:
```json
{
  "matcher": "Bash",
  "hooks": [{ "type": "command", "command": "/path/to/guard.sh", "timeout": 5 }]
}
```

freeze-guard.sh reference: ~/.claude/hooks/freeze-guard.sh (91 lines)
- reads stdin once (INPUT=$(cat))
- checks state file existence (fast path)
- extracts tool_name and tool_input from JSON
- normalizes paths with python3 os.path.realpath
- denies with jq -n JSON output

Script pattern (from Phase 7):
- shebang + set -euo pipefail
- Apache 2.0 header
- --help flag
- POSIX tools (bash 3.2+)

Content sections:
1. Title: `/careful` — Destructive Command Guard via PreToolUse Hook
2. Opening: explains this is a session-scoped safety net that physically blocks destructive Bash commands at the hook level. Uses the same on-demand hook pattern as `/freeze` — permanently registered, activated/deactivated via state file.
3. "When to Use This Skill" table: working near production, refactoring with risky deletes, database migration sessions, Kubernetes operations, pair programming safety
4. Usage:
   ```
   /careful on        # activate destructive command blocking
   /careful off       # deactivate
   /careful status    # check current state
   ```
5. Blocked Commands (what the hook detects in Bash tool_input.command):
   - `rm -rf` (recursive force delete — allows `rm` without `-rf`)
   - `git push --force` and `git push -f` (force push)
   - `git reset --hard` (destructive reset)
   - `DROP TABLE` and `DROP DATABASE` (SQL destructive operations, case-insensitive)
   - `kubectl delete` (Kubernetes resource deletion)
   - `docker system prune` (Docker cleanup)
   The hook checks the command string for these patterns using grep. It does NOT block:
   - `rm` without `-rf` flag
   - `git push` without `--force`/`-f`
   - `git reset` without `--hard`
   - SELECT/INSERT/UPDATE SQL
   - `kubectl get`/`kubectl describe`
6. Implementation — On `/careful on`:
   ```bash
   STATE_FILE="$HOME/.claude/state/careful-state.json"
   mkdir -p "$(dirname "$STATE_FILE")"
   TMP_FILE=$(mktemp "${STATE_FILE}.tmp.XXXXXX")
   cat > "$TMP_FILE" <<EOF
   {
     "active": true,
     "activated_at": "$(date -u +%Y-%m-%dT%H:%M:%SZ)"
   }
   EOF
   mv "$TMP_FILE" "$STATE_FILE"
   echo "Careful mode active: destructive commands will be blocked by PreToolUse hook"
   ```
7. Implementation — On `/careful off`:
   ```bash
   rm -f "$HOME/.claude/state/careful-state.json"
   echo "Careful mode deactivated: all commands allowed"
   ```
8. Implementation — On `/careful status`:
   ```bash
   STATE_FILE="$HOME/.claude/state/careful-state.json"
   if [ -f "$STATE_FILE" ]; then
     AT=$(jq -r '.activated_at // "(unknown)"' "$STATE_FILE" 2>/dev/null)
     echo "Careful mode ACTIVE (since $AT)"
   else
     echo "Careful mode inactive"
   fi
   ```
9. On-Demand Hook Registration section:
   - Same pattern as freeze: `careful-guard.sh` registered permanently in `~/.claude/settings.json`
   - Matcher: `Bash` (not Edit|Write like freeze — careful guards Bash commands)
   - Link to `references/hook-setup.md` for detailed registration instructions
10. Gotchas section with 4 named ### patterns:
    - **Session Restart After Registration**: Same as freeze — hooks loaded at session start. First-time registration requires restart. State file changes take effect immediately.
    - **Pattern Matching Limitations**: The hook uses string matching on the command text, not AST parsing. A command like `echo "rm -rf"` (string literal, not execution) will also be blocked. This is intentional — false positives are safer than false negatives for destructive operations.
    - **Pipe and Subshell Detection**: The hook checks the full command string. `echo foo | xargs rm -rf` will be caught because `rm -rf` appears in the string. However, commands built dynamically at runtime (e.g., `eval "$CMD"` where CMD contains `rm -rf`) cannot be detected — the hook sees `eval "$CMD"`, not the expanded value.
    - **Careful vs Freeze Coexistence**: Both hooks can be active simultaneously. They guard different tools (freeze: Edit/Write, careful: Bash) and different state files. No conflict — they are independently activated and deactivated.
11. Additional Resources: link to `references/hook-setup.md`

**File 2: `skills/careful/scripts/careful-guard.sh`**

Bash script — PreToolUse hook that blocks destructive Bash commands.
Follow the freeze-guard.sh pattern exactly for structure:

```bash
#!/bin/bash
# careful-guard.sh — PreToolUse hook: block destructive Bash commands
set -euo pipefail

STATE_FILE="$HOME/.claude/state/careful-state.json"
INPUT=$(cat)

# Fast path: no state file = inactive
if [ ! -f "$STATE_FILE" ]; then
  echo "{}"
  exit 0
fi

# Only guard Bash tool
TOOL_NAME=$(echo "$INPUT" | jq -r '.tool_name // ""' 2>/dev/null || echo "")
if [ "$TOOL_NAME" != "Bash" ]; then
  echo "{}"
  exit 0
fi

# Extract command from tool_input
COMMAND=$(echo "$INPUT" | jq -r '.tool_input.command // ""' 2>/dev/null || echo "")
if [ -z "$COMMAND" ]; then
  echo "{}"
  exit 0
fi

# Check for destructive patterns (case-insensitive where appropriate)
BLOCKED=""
if echo "$COMMAND" | grep -qE 'rm\s+(-[a-zA-Z]*r[a-zA-Z]*f|[a-zA-Z]*f[a-zA-Z]*r)'; then
  BLOCKED="rm -rf (recursive force delete)"
elif echo "$COMMAND" | grep -qE 'git\s+push\s+.*(-f|--force)'; then
  BLOCKED="git push --force (force push)"
elif echo "$COMMAND" | grep -qE 'git\s+reset\s+--hard'; then
  BLOCKED="git reset --hard (destructive reset)"
elif echo "$COMMAND" | grep -qiE 'DROP\s+(TABLE|DATABASE)'; then
  BLOCKED="DROP TABLE/DATABASE (SQL destructive operation)"
elif echo "$COMMAND" | grep -qE 'kubectl\s+delete'; then
  BLOCKED="kubectl delete (Kubernetes resource deletion)"
elif echo "$COMMAND" | grep -qE 'docker\s+system\s+prune'; then
  BLOCKED="docker system prune (Docker cleanup)"
fi

if [ -n "$BLOCKED" ]; then
  jq -n \
    --arg blocked "$BLOCKED" \
    --arg cmd "$COMMAND" \
    '{
      hookSpecificOutput: {
        hookEventName: "PreToolUse",
        permissionDecision: "deny",
        permissionDecisionReason: ("CAREFUL GUARD: Blocked \($blocked). Command: \($cmd). Run /careful off to disable protection.")
      }
    }'
  exit 0
fi

# Not a destructive command — allow
echo "{}"
exit 0
```

Key implementation notes:
- The rm regex matches `-rf`, `-fr`, `-rfi`, etc. (any flag combo containing both r and f)
- git push force detection handles both `-f` and `--force` flags
- DROP TABLE/DATABASE is case-insensitive (grep -i)
- All other patterns are case-sensitive

Make executable: `chmod +x skills/careful/scripts/careful-guard.sh`
