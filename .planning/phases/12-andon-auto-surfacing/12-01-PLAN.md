RED phase: Write tests/test_gotcha_surfacer.py with all behavior tests above. Tests import from gotcha_surfacer (which does not exist yet) and MUST fail.

GREEN phase: Create hooks/gotcha_surfacer.py with:

1. ConfidenceLevel enum: EXACT (score=1.0), PARTIAL (score range 0.5-0.8 based on word overlap), CATEGORY (score range 0.2-0.4)

2. MatchResult dataclass: gotcha (GotchaEntry), confidence (ConfidenceLevel), score (float)

3. match_gotchas(error_text: str, gotchas: list[GotchaEntry]) -> list[MatchResult]:
   - For each GotchaEntry, check pattern field against error_text:
     a. EXACT: The pattern text (lowercased, normalized whitespace) appears as a substring of the error text (lowercased, normalized whitespace). Score = 1.0.
     b. PARTIAL: Extract significant words (3+ chars, not stopwords) from the pattern. If 40%+ of pattern words appear in error text, it is a partial match. Score = 0.5 + 0.3 * (matching_words / total_words), capped at 0.8.
     c. CATEGORY: If the error text contains any of the Gotcha's tags (case-insensitive whole-word match), score = 0.3. If it matches severity keyword, score = 0.2.
   - Return matches sorted descending by score. Skip entries with no match.

4. format_surfaced_gotchas(matches: list[MatchResult]) -> str:
   - Format each match as: "[Gotcha {id}] {name} ({confidence_label}, score={score:.1f}): {prevention}"
   - Separate entries with newline.
   - Prefix with "[GOTCHA_MATCH] Matching Gotchas found:" header line.
   - Return empty string if matches is empty.

5. surface_gotchas(error_text: str, gotchas_dir: Path, packs_dir: Path | None = None) -> str:
   - Calls load_gotchas(gotchas_dir, packs_dir=packs_dir) to get registry.
   - Calls match_gotchas(error_text, gotchas).
   - Calls format_surfaced_gotchas(matches).
   - Returns the formatted string (empty if no matches).
   - Catches GotchaValidationError and logs a warning, returning empty string (surfacing should never block ANDON).

IMPORTANT: The `pattern` field in GotchaEntry is a prose description (2-4 sentences), NOT a regex. Matching is keyword/substring based, not regex-based. This is deliberate because the YAML pattern fields contain natural language descriptions of anti-patterns, not regex patterns.

Use only stdlib imports plus the existing gotcha_registry module (no new dependencies).
