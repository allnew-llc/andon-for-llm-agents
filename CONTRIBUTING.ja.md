# ANDON for LLM Agents へのコントリビューション

コントリビューションへのご関心をいただきありがとうございます！このドキュメントでは、本プロジェクトへのコントリビューション手順を説明します。

## コントリビューション方法

### Issue の報告

- バグ報告や機能提案には GitHub Issues を使用してください。
- 再現手順、期待する動作、実際の動作を含めてください。
- セキュリティ脆弱性については、下記の[セキュリティ](#セキュリティ)を参照してください。

### Pull Request

1. リポジトリをフォークし、フィーチャーブランチを作成してください。
2. 明確で説明的なコミットメッセージで変更を行ってください。
3. 必要に応じてテストを追加または更新してください。
4. テストスイートを実行してください：`python3 -m pytest tests/ -q`
5. 変更内容とその理由を説明する PR を提出してください。

### 関心のある分野

- 他の言語/フレームワーク向けの追加の失敗分類パターン
- Claude Code 以外の IDE 統合（Cursor、Windsurf 等）
- 翻訳（現在は英語 + 日本語）
- Knowledge Pack のコントリビューション（[CONTRIBUTING-PACKS.md](./CONTRIBUTING-PACKS.md) を参照）
- ANDON/Kaizen を LLM コーディングワークフローに適用したケーススタディ

## コードスタイル

- Python：PEP 8 に準拠。型アノテーションを使用。
- Shell：`set -euo pipefail` を使用。変数をクォート。
- Markdown：ソース内では1文1行（差分の可読性のため）。

## テスト

```bash
# 全テストを実行
python3 -m pytest tests/ -q

# 詳細出力で実行
python3 -m pytest tests/ -v
```

すべての PR は既存のテストスイートを通過する必要があります。新機能にはテストを追加してください。

## セキュリティ

セキュリティ脆弱性を発見した場合、公開 Issue を作成**しないでください**。代わりに、GitHub のセキュリティアドバイザリ機能を使用して非公開で報告するか、メンテナーに直接連絡してください。48 時間以内に対応いたします。
